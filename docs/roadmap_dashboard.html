§4 Three-way, by scenario — where each engine wins

The visual examples split into four scenarios, and the ranking flips between them. No engine wins everywhere.

Organic (spot · bunny · rocker) — quad-cover ties/beats QuadriFlow

With quad-cover: irregular ours 1–4% · QF 3–4% · AR 6–15% (we win spot & rocker, tie bunny); CV ours 0.15–0.23 · QF 0.14–0.17 · AR 0.41–0.48; median ours 84–85° · QF 82° · AR 75–80°. The seamless-UV isoline extractor closed the organic gap that the collapse-paradigm integer path (7–13%) could not, and it beats AutoRemesher outright.

CAD / sharp (cube) — everyone struggles; ours now safe

On a flat-faced cube (900-quad target): QF nails 89° angles but tears the mesh — 376 defects; AutoRemesher is clean (0 defects, 90°) but CV 1.39. Our integer path used to break (312 defects, 61% irregular) — now the flatness routing (§6·E) sends flat patches to field-aligned, so it's manifold at 24% irregular. Flat CAD is still the hardest regime for all field-based extractors.

Adaptivity — only ours

Curvature-adaptive sizing spends polygons where the surface bends; it beats our own uniform on 4 / 5 models (§3). QuadriFlow and AutoRemesher are uniform-only — this axis is ours alone.

Robustness — ours (field-aligned)

Our field-aligned path returns 0 topological defects on the cube where QuadriFlow tears it (376). AutoRemesher is clean too but double-frees in teardown on CAD (we patched our harness to survive it). The always-manifold guarantee is a real edge for production use.

§5 What the integer extractor still misses — and how quad-cover closed it

Resolved by §6·I (quad-cover)

The four gaps below are the integer extractor's. The seamless-UV quad-cover extractor closes gaps 1–3 structurally: irregular 9–14%→1–4%, edge CV 0.29–0.42→0.15–0.28, median 74–79°→81–85°. The integer path is kept as a manifold-guaranteed alternative; the notes below record why the collapse paradigm plateaued.

The integer extractor is benchmarkable (quad_method="integer"), runs end-to-end on the whole corpus (the OOM is fixed) and was best-of-ours on singularity count before quad-cover. Its two residual gaps are topological, not geometric — which is why the position pass helped angle more than CV. Integer vs QuadriFlow at the ~3000-quad target: irregular 9–14% vs 1–4% · edge CV 0.29–0.42 vs 0.14–0.19 · median 74–79° vs 82°.

1 · Fewer singularities closing

irregular 7–13% vs QuadriFlow 1–4% — down from 9–14% now that the dipole canceller (§6·B) ships (spot 9→7, cheb 11→9, bunny 14→13). The collapse still quantizes the grid into val3/val5 dipoles + val3 T-junctions; the canceller removes the dipoles a neighbour can absorb (~60% val3 / ~30% val5). The remaining excess is subdivision T-junctions — the next lever is upstream (§6·C: 2nd subdivide + FixFlipSat → fewer |diff|>1 edges), not the coarse-face read, which was proven a no-op (§6·A).

2 · Uniform quads pass added

edge CV 0.29–0.42 vs QuadriFlow 0.14–0.19. A tangential-Laplacian position pass now relaxes the extracted vertices within their tangent plane (relaxPositions, commit 118aeaa): spot median 64→70°, dev 0.56→0.54%. But CV plateaus near 0.63 regardless of iterations — the unevenness is in the cells (topology), not the vertex positions, so smoothing can't reach QuadriFlow's CV. Gated on gap 1.

3 · Straighter angles

median 74–79° vs QuadriFlow 82°. Mostly downstream of gaps 1 & 2 — irregular vertices skew their neighbours and non-uniform quads shear. The position pass already bought back a few degrees; closing the singularity count recovers the rest for free.

4 · Robustness fixed

The integer path used to OOM-kill on cheburashka / rocker-arm at coarse targets (a subdivide + merge-path runaway). Now bounded three ways (subdivide cap, merge-path cap, flip-repair validate-and-discard) with a regression test — commit ff2b866. Output on those coarse grids is sparse-but-valid, never a crash. Sharp-crease / open-boundary handling for CAD input is the remaining robustness item.

A · Multi-resolution extraction — DEAD END (proven) — the plan was to read faces off QuadriFlow's coarsened edge graph. Built it and measured: byte-identical to the single-level path (3398 == 3398 compact triangles). Reason: DownsampleEdgeGraph only contracts zero-diff edges, but subdivision split-vertices are joined by |diff|=1 edges, so the coarse faces are exactly the fine non-degenerate faces. QuadriFlow's low count comes from its upstream 2nd-subdivide + FixFlipSat + position-solve, not the coarse read. Reverted. no-op, reverted B · Valence-3/5 dipole canceller — DONE — measured the irregulars: ~60% val3, ~30% val5 (dipoles). The extractor is Qt-free and Geogram-free (the exploragram include is vestigial/dead); the seamless UV it needs comes from Geogram's quad_cover in parameterizer.cpp. It does run headless (--input/--target-quads), and the parameterize+extract core is Qt-free, so a small headless harness (over bundled Geogram/TBB/Eigen) can be built for the benchmark — planned as a second reference alongside QuadriFlow.
