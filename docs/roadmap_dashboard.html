§4 What the benchmark taught us

Robust on hard geometry

On a subdivided cube QuadriFlow tears the mesh apart (598 boundary edges) and rounds the corners; ours stays clean and crisp. A genuine strength on box/CAD geometry — though on smooth organic models QuadriFlow is the more robust one.

Solve + watertight extraction

The global integer solve (min-cost flow) cancels the spurious singularities, and QuadriFlow's BuildTriangleManifold + FixValence turn that grid into a watertight, manifold, all-quad mesh — beating the current default extractor on irregular % across the corpus (spot 25% vs 44%, rocker-arm 24% vs 68%). It does not yet beat QuadriFlow: the zero-diff collapse quantizes the clean grid, multiplying its 44 true singularities ~9× into val3/val5 dipoles (the pairing and field are both clean — round 2). Closing it needs the multi-res flip-repair layer — Phase 5.

Adaptivity now works

A per-vertex spacing (local density) stops the extractor over-merging adaptive input — the count no longer collapses (8× more quads) and adaptive sizing beats our own uniform on 4 / 5 models. Beating QuadriFlow absolutely waits on Phase 4.

§5 What's missing to match QuadriFlow

The integer extractor is benchmarkable (quad_method="integer"), runs end-to-end on the whole corpus (the OOM is fixed) and is best-of-ours on singularity count. Two of the four gaps are now addressed; the two that remain are topological, not geometric — which is why the position pass helped angle more than it helped CV. Integer vs QuadriFlow at the ~3000-quad target: irregular 9–14% vs 1–4% · edge CV 0.29–0.42 vs 0.14–0.19 · median 74–79° vs 82°.

1 · Fewer singularities closing

irregular 7–13% vs QuadriFlow 1–4% — down from 9–14% now that the dipole canceller (§6·B) ships (spot 9→7, cheb 11→9, bunny 14→13). The collapse still quantizes the grid into val3/val5 dipoles + val3 T-junctions; the canceller removes the dipoles a neighbour can absorb (~60% val3 / ~30% val5). The remaining excess is subdivision T-junctions — the next lever is upstream (§6·C: 2nd subdivide + FixFlipSat → fewer |diff|>1 edges), not the coarse-face read, which was proven a no-op (§6·A).

2 · Uniform quads pass added

edge CV 0.29–0.42 vs QuadriFlow 0.14–0.19. A tangential-Laplacian position pass now relaxes the extracted vertices within their tangent plane (relaxPositions, commit 118aeaa): spot median 64→70°, dev 0.56→0.54%. But CV plateaus near 0.63 regardless of iterations — the unevenness is in the cells (topology), not the vertex positions, so smoothing can't reach QuadriFlow's CV. Gated on gap 1.

3 · Straighter angles

median 74–79° vs QuadriFlow 82°. Mostly downstream of gaps 1 & 2 — irregular vertices skew their neighbours and non-uniform quads shear. The position pass already bought back a few degrees; closing the singularity count recovers the rest for free.

4 · Robustness fixed

The integer path used to OOM-kill on cheburashka / rocker-arm at coarse targets (a subdivide + merge-path runaway). Now bounded three ways (subdivide cap, merge-path cap, flip-repair validate-and-discard) with a regression test — commit ff2b866. Output on those coarse grids is sparse-but-valid, never a crash. Sharp-crease / open-boundary handling for CAD input is the remaining robustness item.

§6 The plan to close the gap

Four tasks resolved (A dead-end, B shipped, C ceiling, F benchmarked); the findings reshaped the rest. The big surprise: benchmarking AutoRemesher (F) showed QuadCover-in-practice is no better than us — QuadriFlow is the one real target (G). Both references are MIT.

A · Multi-resolution extraction — DEAD END (proven) — the plan was to read faces off QuadriFlow's coarsened edge graph. Built it and measured: byte-identical to the single-level path (3398 == 3398 compact triangles). Reason: DownsampleEdgeGraph only contracts zero-diff edges, but subdivision split-vertices are joined by |diff|=1 edges, so the coarse faces are exactly the fine non-degenerate faces. QuadriFlow's low count comes from its upstream 2nd-subdivide + FixFlipSat + position-solve, not the coarse read. Reverted. no-op, reverted B · Valence-3/5 dipole canceller — DONE — measured the irregulars: ~60% val3, ~30% val5 (dipoles). Added a valence-improving edge-rotation pass to FixValence: re-draw an interior edge along a hexagon diagonal, moving valence off the 3/5 pair onto a neighbour, accepted only when it strictly lowers local irregulars and stays manifold (monotone). Irregular −27% at native density (spot 20.2→16.8%); at 3000-quad: spot 9→7, fandisk 10→9, rocker 10→8, cheb 11→9, bunny 14→13. Regression-tested, ASan-clean. shipped · a83ac1f C · Reduce split T-junctions — CEILING REACHED — the ~60% val3 excess is subdivision T-junctions. Tried two levers, both no-ops: (1) QuadriFlow's unit-grid FixFlipSat reduced spot only 16.8→16.2% at 2.3 s/mesh; (2) a val-3 T-junction edge-collapse (AutoRemesher removeSingleEndpoints dual) found zero legal monotone collapses. These Ts sit at genuine field-singularity mismatches that no local, manifold, monotone operator can remove. Paradigm-inherent — the real fix is F. exhausted D · Surface-projected position relaxation — fold the Laplacian pass onto the source surface via the existing BVH (src/accel) instead of a per-vertex tangent plane. Marginal CV, protects fidelity. extend · relaxPositions E · Sharp-crease + open-boundary handling — detect source creases (sharpEdgeDegrees) and pin/align extracted vertices before relaxation, so CAD input (fandisk) keeps its edges. feature F · Benchmarked AutoRemesher (QuadCover) — DONE, and it reframes everything — built a Qt-free headless harness over its QuadCover core (Geogram quad_cover seamless-UV solver + isoline quadextractor.cpp) and added it as a 2nd benchmark reference. The theory did NOT hold in practice: QuadCover's isoline tracing is T-junction-free in principle, but AutoRemesher's actual pipeline (isotropic remesh → frame field → quad_cover) lands at irregular 6–15% (spot 6, fandisk 14, cheb 15, bunny 12) — comparable to our integer extractor (7–13%) — with the worst edge CV of all (0.41–0.56). So the seamless-UV paradigm is not a guaranteed path to parity, and we are already competitive with AutoRemesher. shipped · 7aba362 G · Close on QuadriFlow, the real gold standard — the benchmark's verdict: QuadriFlow alone hits 1–4% irregular + CV 0.14–0.19; both ours and AutoRemesher trail it. QuadriFlow's edge over both is its upstream integrity (2nd subdivide + FixFlipSat + position solve → fewer |diff|>1 edges → fewer splits), not a magic extractor. That upstream-integrity work — deferred as marginal in C at native density — is worth revisiting at the coarse benchmark density where splits dominate. the real target

Reference: autoremesher / QuadCover MIT · port

huxingyi/autoremesher (Dust3D) is MIT. Its core algorithm is QuadCover (quadextractor.cpp) — isoline tracing over a seamless UV, studied in full (§6·F). The extractor is Qt-free and Geogram-free (the exploragram include is vestigial/dead); the seamless UV it needs comes from Geogram's quad_cover in parameterizer.cpp. It does run headless (--input/--target-quads), and the parameterize+extract core is Qt-free, so a small headless harness (over bundled Geogram/TBB/Eigen) can be built for the benchmark — planned as a second reference alongside QuadriFlow.
