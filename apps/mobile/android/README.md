# CyberRemesher — Android shell (task 8.5)

> **UNVERIFIED / best-effort.** Requires the Android SDK (API 26+) + NDK and the
> cross-compiled `libcyber_capi` for each ABI. Not built in CI; not wired into
> the repo CMake tree. The `cpp/CMakeLists.txt` here is the **app's own NDK
> build script** (invoked by Gradle/`externalNativeBuild`), independent of the
> engine's CMake. Every source file starts with `UNVERIFIED:`.

A thin Kotlin/Jetpack-Compose shell that talks to the engine's **C ABI**
(`capi/cyber_capi.h`) through a small JNI shim (`cpp/cyber_jni.cpp`). Like the
iPadOS shell it adds **no** engine logic — stroke recognition, chording and tool
routing live in C++ (`src/app`).

## What this shell demonstrates (8.5)

- **Touch feed** — `EngineSurfaceView.onTouchEvent` expands historical
  (batched) motion samples and forwards them, normalized to `0..1`, into the
  engine session via `CyberEngine.injectStroke`.
- **Lifecycle autosave** — `MainActivity`/`EngineSession` observe the
  `Lifecycle` (`ON_PAUSE`/`ON_STOP`) and snapshot + persist the document before
  the process can be backgrounded/killed.
- **Vulkan/GLSurface host** — `EngineSurfaceView` is a `SurfaceView`; its
  `Surface` is handed to native (`ANativeWindow`) and attached to the engine
  render backend. The design mandates **Vulkan** on Android (no OpenGL backend);
  a GLSurface path is a fallback only.

It renders the **8.6** shared UI (stage switcher, progress/cancel overlay with
atomic commit, Action Gallery toolbar, in-app log) from `../shared/*.json`,
which are copied into `assets/` at build time.

## Building

Scaffolding only. To make it runnable:

1. Cross-compile the engine + `capi/` for the target Android ABIs
   (`arm64-v8a`, `x86_64`) → `libcyber_capi.so` per ABI.
2. Drop `cyber_capi.h` + the `.so`s where `cpp/CMakeLists.txt` expects them (see
   the `CYBER_CAPI_DIR` variable), or point `externalNativeBuild` at the engine
   build tree.
3. `./gradlew :app:assembleDebug` (needs the Android SDK/NDK + a Gradle wrapper
   generated by `gradle wrapper`).

## Assumed ABI

The JNI shim targets the **same assumed session ABI** the Swift package
documents (`swift/README.md` → "Assumed C ABI contract"): `cyber_session_*`,
`cyber_mesh_*`, `CyberStrokeSample`. If `capi/` lands with different spellings,
update `cyber_jni.cpp` only — the Kotlin surface does not change.
